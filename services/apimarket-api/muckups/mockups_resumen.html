Mockups para Microservicio eqhuma

Manteniendo el formato y colores del microservicio Palencia

Descripción del Proyecto

Este documento presenta los mockups desarrollados para el microservicio que replicará la funcionalidad de eqhuma, manteniendo el mismo formato y colores que el microservicio Palencia desarrollado anteriormente. Los mockups incluyen todas las pantallas principales necesarias para la interacción del usuario con la plataforma.

1. Inicio de Sesión

Pantalla de inicio de sesión para usuarios ya registrados. Mantiene la estética minimalista y los colores corporativos de Palencia.

Características: Formulario de ingreso, opción de recordar credenciales, enlace para recuperación de contraseña y registro de nuevos usuarios.

Ver pantalla completa
2. Página Principal del Marketplace

Pantalla principal que muestra el catálogo de APIs disponibles, con filtros y opciones de búsqueda.

Características: Buscador, filtros por institución y tipo, tarjetas de APIs organizadas en cuadrícula, indicadores de uso y rendimiento.

Ver pantalla completa
3. Detalle de API

Pantalla que muestra información detallada sobre una API específica, incluyendo su documentación, ejemplos de uso y opciones de suscripción.

Características: Descripción, endpoint, parámetros de solicitud, ejemplos de código, respuestas esperadas y códigos de estado.

Ver pantalla completa
4. Documentación Técnica

Página con la documentación técnica general para desarrolladores, incluyendo guías de integración, autenticación y manejo de errores.

Características: Guías por secciones, ejemplos de código, detalles de autenticación, manejo de errores y rate limiting.

Ver pantalla completa
5. Dashboard de Usuario

Panel de control para usuarios que muestra estadísticas de uso, API keys, suscripciones activas y detalles de consumo por API.

Características: Estadísticas generales, gráficos de uso, administración de API keys, detalles de consumo por API y gestión de suscripciones.

Ver pantalla completa

Conclusiones y Próximos Pasos

Los mockups presentados demuestran cómo se puede replicar la funcionalidad de eqhuma manteniendo el formato y colores del microservicio Palencia. Estos diseños proporcionan una base sólida para el desarrollo frontend del microservicio.

Próximos Pasos

  1. Desarrollar las plantillas HTML en componentes React/Vue para implementación frontend
  2. Conectar con la API backend que se desarrollará según el informe técnico
  3. Implementar la lógica de autenticación y autorización
  4. Desarrollar el sistema de visualización de documentación interactiva
  5. Integrar con el microservicio existente de Palencia
